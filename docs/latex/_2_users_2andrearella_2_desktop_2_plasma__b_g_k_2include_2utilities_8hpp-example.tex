Storage of utility functions and concepts for the Plasma\+\_\+\+BGK project.\+In this file, we define various utility functions and concepts that are used throughout the Plasma\+\_\+\+BGK project. These utilities include


\begin{DoxyItemize}
\item \begin{DoxySeeAlso}{See also}
Mesh\+Container concept to check if a type is a range and holds mesh components.
\end{DoxySeeAlso}

\item \begin{DoxySeeAlso}{See also}
Floating\+Point concept to check if a type is a floating point type.
\end{DoxySeeAlso}

\item \begin{DoxySeeAlso}{See also}
Spacing\+Function concept to define a callable type for mesh spacing.
\end{DoxySeeAlso}

\item \begin{DoxySeeAlso}{See also}
Mesh\+Nature enum class to define the nature of a mesh (space or velocity).
\end{DoxySeeAlso}

\item \begin{DoxySeeAlso}{See also}
QUICK\+\_\+coefficients\+\_\+p Function to compute QUICK coefficients for finite volume meshes.
\end{DoxySeeAlso}

\item \begin{DoxySeeAlso}{See also}
QUICK\+\_\+coefficients\+\_\+n Function to compute QUICK coefficients for finite volume meshes.
\end{DoxySeeAlso}
These utilities are essential for ensuring type safety and providing common functionality across different mesh implementations.
\end{DoxyItemize}

Throws a std\+::runtime\+\_\+error in debug builds if the condition is false. In release builds, logs a warning message instead.

This macro checks the expression {\ttfamily expr}.
\begin{DoxyItemize}
\item If {\ttfamily expr} evaluates to false in debug mode ({\ttfamily NDEBUG} {\bfseries{not}} defined), it throws an exception with the provided message {\ttfamily msg}.
\item If in release mode ({\ttfamily NDEBUG} defined), it prints a warning to {\ttfamily std\+::cerr} but does not throw.
\end{DoxyItemize}
